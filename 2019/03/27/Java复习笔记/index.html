Java 复习笔记

1. 代码块

  • 局部代码块: {}， 相当于一个局部的代码, 生命周期在代码块内。注意是在method里面的{}
  • 构造代码块: 也是{}， 不过这里的{}存在于方法外，类内。执行先于构造函数
  • Static代码块: static{}, 只在类被加载的时候执行，且只执行一次

2. static, this

  • this 指针: 就是object指向自己的指针，只在创建了object之后自动存在此object中。类中没有this指针。 所以static的method中不可以用this指针。
  • static 只能访问static的成员变量

3. 继承

  • public 和 private 其实都会被继承， 但是子类不可直接访问父类的私有attributes。子类可以通过父类的方法访问到父类的private attributes. 但是父类的私有方法子类不可能使用
  • 子类默认调用了父类的无参构造函数super()，我们也可以手动使用带参构造函数 super(args)替换掉默认调用.
  • 方法重写:
    • 直接重写
    • 在重写方法中调用super.重写方法(args) 写在第一句先继承父类方法，再加上新功能达到加强的效果。

4. final

  • 用在类前, 最终类， 无法被继承
  • 用在方法前， 最终方法， 无法被override
  • 用在变量前，最终变量，即无法改变的变量， 即常量

5. 基本类型和引用类型

  • 基本类型: byte, char, short, int, long, float, double。 不需要new
  • 引用类型: 类似于C的指针, 在JAVA中有 类，数组，接口，枚举等。需要new

6. 多态

  • 父类无法使用子类独有的方法和成员。但是父类可以直接使用子类重写的方法，使其变得更强大。

7. Abstract

  • abstract 类不能创建对象
  • 有abstract方法的类一定是abstract类
  • 如果abstract类中存在abstract方法，继承其的子类必须实现抽象方法。

8. 接口

  • interface 创建接口, implements 实现接口
  • 只存在抽象方法
  • 变量前一定是public static final (共享的常量)
  • 接口没有构造方法

9. 抽象类和接口

  • 抽象类的子类和抽象类是is a的关系，但是实现接口的类与接口是like a的关系
  • 接口就是往某个类里面加上功能

10. extends 和 implements

  • JAVA的规则是单继承，但是可以实现多个接口

11. Package, import, class 的位置

  • Package 在最开头, import 在package下方, class 在package 下方
